4

JWT Auth Flow

The highest-value interview section.

>

Register

POST /api/auth/register creates the user, creates an empty CareerProfile, returns the access token, and sets the refresh cookie.

Login

Checks the password with comparePassword(), updates lastLogin, and issues new tokens.

Refresh

/api/auth/refresh verifies the cookie token and checks it matches the token stored on the user record.

Token Lifetimes

Access token: 15m. Refresh token: 7d.

authMiddleware

Checks the Bearer token, verifies JWT, loads the user, and rejects inactive or missing accounts.

Security Answer

Long-lived refresh tokens are safer in httpOnly cookies than in browser storage accessible to scripts.

JWT internals
  • JWT payload contains userId, email, and issuedAt.
  • JWT secret is loaded from process.env.JWT_SECRET and used in sign/verify.
  • Refresh flow uses refreshToken in cookie; verified with same secret and matched to user value from DB, preventing reuse after logout.
  • When refreshing, the backend rotates the refresh token token every time, then updates the user record with new refresh data.
DB auth fields
  • User persists passwordHash, refreshToken, lastLogin and isActive status.
  • Logout clears refresh token server-side and removes cookie to invalidate session immediately.
  • Revocation supports a stale refresh token check; users from soft-deleted accounts are blocked at user load.

Data Models

Remember the relationship map first, hooks second.

>
Simple relationship sentenceOne User owns one CareerProfile and can own many Goal and Assessment documents.

User

Identity, auth, refresh token, and lightweight profile data. Password hashing happens in a pre-save hook.

CareerProfile

The richest model. Stores deep career data, embedded experience and education, resume content, cover letters, and insights.

Goal

Tracks planning, progress, milestones, category, priority, and completion timestamps.

Assessment

Stores review quiz results, AI analysis, strengths, and improvement plans.

Database design details
  • User has unique index on email and fields: password (hashed), refreshToken, lastLogin, isActive.
  • CareerProfile uses nested arrays for experience/education and computed completionScore plus text-heavy fields for AI recommendations.
  • Goal supports query by user, status, dueDate; updates run via controller patch for safe increments.
  • Assessment stores question responses, score metrics, and AI analysis subobject for “strengths”/“improvements”.
  • Soft delete pattern in user account deletion: remove credentials + clear related CV/goals/assessments references with cascading remove in controller.

Gemini Integration

Where AI lives and how it gets saved.

>

Where Gemini Runs

All AI calls start in src/services/ai-service.js and default to gemini-2.5-flash.

JSON Cleanup

The service removes markdown code fences before parsing AI JSON for insights and quiz questions.

Fallbacks

If Gemini fails, the frontend can still provide fallback insights, quiz questions, and other default content.

Production tradeoffThis frontend-only AI pattern is great for learning but not ideal for production. A real production version should proxy AI calls through the backend to protect API keys and centralize control.
